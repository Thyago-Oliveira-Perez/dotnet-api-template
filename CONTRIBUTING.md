Harassment, discrimination, or disrespectful behavior of any kind will not be tolerated.

## How Can I Contribute?

### Reporting Bugs

If you encounter a bug:

* Search existing issues to avoid duplicates
* Open a new issue with:

  * A clear and descriptive title
  * Steps to reproduce the issue
  * Expected vs. actual behavior
  * Relevant logs, screenshots, or stack traces (if applicable)

Clear bug reports help issues get resolved faster.

### Submitting Pull Requests

Before opening a pull request:

* Fork the repository
* Create a feature branch from the default branch (`main` or `master`)
* Ensure your changes are focused and scoped
* Add or update tests when applicable
* Update documentation if behavior or APIs change

Pull requests should:

* Have a clear description of *what* was changed and *why*
* Reference related issues when applicable
* Be small and reviewable whenever possible

## Development Guidelines

General guidelines:

* Keep code readable and maintainable
* Favor clarity over cleverness
* Follow existing project structure and conventions
* Avoid introducing unnecessary abstractions
* Write tests for new logic when appropriate

Consistency matters more than personal preference.

---

## Commit Messages

Use clear and meaningful commit messages.

Recommended format:

* Short summary in the first line (imperative mood)
* Optional detailed description in the body

Examples:

* `Fix incorrect date parsing in report generation`
* `Add validation for empty input values`
